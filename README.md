This repository contains an implementation of a Maximum Likelihood Estimation (MLE) framework to model a 17-bit Arbiter Physical Unclonable Function (PUF) containing missing latent data.

## Overview
The objective is to estimate the physical parameters (weights and biases) of a 17-bit Arbiter PUF from a set of Challenge-Response Pairs (CRPs). The 9th bit of every challenge is latent (hidden). Because the objective function is non-convex with respect to simultaneously finding the PUF parameters and the latent bits, the problem is solved using an Expectation-Maximization (Hard-EM) alternating optimization pipeline.

## Implementations

### 1. Unconstrained Prior (`my_latent`)
* **Assumption:** The latent variable is unconstrained (flat prior of $P = 0.5$).
* **Mechanism:** The M-step fits the 17-bit model, and the E-step updates latent bits strictly to minimize the 17-bit prediction error.
* **Result:** The model overfits the dataset. It updates the latent bits to force mathematical convergence, yielding 99.99% training accuracy, but fails to capture the underlying hardware parameters.

### 2. Generative Prior Regularization (`my_latent_updated`)
* **Assumption:** The latent variable is deterministically generated by a secondary, hidden 16-bit Arbiter PUF.
* **Mechanism:** The E-step requires a latent bit to maximize the joint probability of both the 16-bit hidden model and the 17-bit final response.
* **Result:** The dual-model constraint acts as a structural regularizer. By restricting latent updates to physically probable states, the EM algorithm converges on the true parameters, yielding 98.40% test accuracy on unseen CRPs.

## Optimization Details
* **Vectorization:** Feature mapping ($\phi(c)$), margin calculations, and log-sigmoid evaluations are fully vectorized using NumPy to handle the 8,000-sample dataset without Python-level loops.
* **Warm-Started Solvers:** The M-step utilizes `sklearn.linear_model.LogisticRegression` with `warm_start=True`, retaining weight matrices across EM iterations to minimize solver overhead.

Overview
This repository contains my code and analysis for compromising a Hamming Physically Unclonable Function (PUF) using machine learning. By mapping the underlying hardware logic into a higher-dimensional space, I built a linear classifier capable of cracking the cryptographic boundary with near-perfect accuracy.

Repository Structure
submit.py: The core Python script. Contains the custom 289-D feature embedding logic (my_map) and the tuned hyperparameters for the SVM (my_params).

Final_Report.pdf: The full write-up. Covers the mathematical proof for the feature expansion, hyperparameter ablation studies, and sample complexity analysis using VC-dimension theory.

How it Works
Standard Arbiter PUFs are relatively straightforward to model, but this Hamming PUF uses non-linear low-level operations (products of partial Hamming weights). To break it using a linear model, I had to expand the 32-bit raw challenges into a custom feature space.

1. Feature Engineering (D = 289)
I expanded the input space to capture all the linear terms and cross-correlations between the even and odd challenge bits. The breakdown:

1 Bias term (to capture the secret threshold)

32 Linear terms (the raw bits)

256 Cross-product terms (even_bit * odd_bit)

2. Model Optimization
I used an sklearn LinearSVC with squared_hinge loss and an L2 penalty. Because we have way more samples than features, I forced the model to use the primal solver (dual=False), which drastically cuts down convergence time.

Results
Peak Accuracy: Hit 99.7% on unseen Challenge-Response Pairs (CRPs).

Sample Efficiency: Proved mathematically and empirically that we only need about 5,000 CRPs to reliably bypass the hardware security (>99.3% accuracy), without needing massive datasets.

Speed: The NumPy vectorization in the feature map keeps execution extremely fast and lightweight.
